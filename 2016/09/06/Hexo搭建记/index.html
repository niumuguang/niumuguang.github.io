发表于 2016-09-06   |   分类于 技术分享   |  

总结网上的使用github+hexo搭建blog时的步奏和踩到的那些坑,力求全面阐述搭建过程和简单介绍hexo发布这篇博客的过程为后来者节省时间.

准备一枚免费域名

免费顶级域名我们可以去Freenom申请,申请之后进入管理页面如下图

进入nameservers管理界面更改dns解析服务器,该解析服务器dnspod官方的,注册后可以进入域名解析界面查询

设置一个域名解析

访问dnspod的网站注册账号,我是用qq直接登录的.

disqus发现国内不能用,所以改用多说同理注册以备后用

开垦一块图床

七牛云直接使用github账号直接登录,进入储存空间,上传blog用到的图片或者下载资源.

Github中创建工程

github账号是必须的了. 登陆后创建一个新的repository以github.io做结尾, 例如: xxxx.github.io,

修改Github Pages为你自己的域名,这个域名要提前申请的;

为什么用github.io作为域名原因?

    1. CDN缓存更方便;
    1. 突破浏览器并发限制;
    1. Cookieless, 节省带宽;
    1. 对于UGC的内容和主站隔离，防止不必要的安全问题;

Hexo 安装

hexo是一款基于Node.js的静态博客框架。所以首先需要安装node.js,访问Node.js官网下载window版安装包,一路next就行.
Hexo官网 安装完成之后进入cmd命令行,输入命令利用 npm 命令即可安装。

npm install hexo-cli -g

content

ok, 一切贮备就绪

hexo server 端口号
